Multicollinearity check¶

For gradient boosted trees, there's generally no strong need to check for multicollinearity because of its robustness. But practically speaking, you still should do some basic checks. For example, if you discover that two variables are 100% the same, then of course there's no point in keeping both. Even if it's 98% correlated, it's usually okay to drop one variable without degrading the overall model. Collinearity may mess up feature importance.

Early stopping¶

In [102]:
# use early stopping to find the number of trees

patience = 5
gbr_cs = ensemble.GradientBoostingRegressor(
    n_estimators=1, 
    max_depth=6, 
    learning_rate=0.05,
    min_samples_split=1, 
    subsample=0.75,
    random_state=42, 
    loss='ls',
    warm_start=True)


best_error = np.inf

for i in range(2000):
    gbr_cs.set_params(n_estimators=i + 1)
    gbr_cs.fit(X_cs_train, y_cs_train)
    val_error = mean_squared_error(y_cs_test, gbr_cs.predict(X_cs_test))
    if val_error < best_error:
        best_error = val_error
    else:
        patience -= 1
        if patience < 0:
            print('Converged at tree #%d' % (i + 1))
            break

Tuning with grid search¶

Tuning model with grid search. Exhaustive search over specified parameter values for an estimator.

Grid search takes a set of possible values for each hyperparameter that should be tuned and evaluates a model trained on each element of the Cartesian product of the sets. It is an exhaustive search that trains and evaluates a model for each possible combination of the hyperparameter values supplied.
